Subindo seu projeto para o GitHub

Objetivo: Aprender como criar um repositório no GitHub e enviar (fazer push) os arquivos de um projeto local para ele.

1. Crie uma conta (se ainda não tiver)

Acesse https://github.com e clique em Sign up para criar uma conta.

2. Criando um novo repositório

  1. Acesse https://github.com e faça login.
  2. Clique no botão "New" ou "Novo repositório".
  3. Escolha um nome (ex: meu-primeiro-repo)
  4. Deixe como público ou privado, como preferir.
  5. Não marque a opção "Add a README" (iremos adicionar localmente).
  6. Clique em Create repository.

3. Conectando seu projeto local ao GitHub

Suponha que você já tenha um projeto HTML versionado com Git (se não tiver, volte para o exercício anterior).

  1. Copie a URL do repositório recém-criado (de preferência em HTTPS).
  2. No terminal, dentro da pasta do seu projeto, digite:
    3. git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git
  3. Garanta que o branch se chame main: 
    git branch -M main
  4. Envie o conteúdo do projeto para o GitHub: 
    git push -u origin main

4. Verifique no navegador

Atualize a página do seu repositório no GitHub. Os arquivos do seu projeto agora estarão disponíveis online!

Comandos utilizados:

  • git remote add origin [url] – Conecta o repositório local ao remoto
  • git branch -M main – Renomeia o branch atual para "main"
  • git push -u origin main – Envia os commits locais para o repositório remoto

Dica:

Você pode usar o GitHub Desktop para fazer isso de forma visual, se preferir uma interface gráfica.
